## 🚨 常见错误及解决

### 错误 1：HTTP 401 Unauthorized

**症状**：
```
[18:30:05] 响应状态: 401 Unauthorized
[18:30:05] 响应内容: Invalid API key
```

**原因**：API Key 错误或无效

**解决方法**：
1. 检查 API Key 是否正确
2. 登录 MiniMax/GLM 平台，重新生成 API Key
3. 更新调试版.html 中的 API Key

---

### 错误 2：HTTP 429 Too Many Requests

**症状**：
```
[18:30:05] 响应状态: 429 Too Many Requests
[18:30:05] 响应内容: Rate limit exceeded
```

**原因**：请求频率过高

**解决方法**：
1. 等待几秒后再试
2. 检查账户余额
3. 升级账户套餐

---

### 错误 3：HTTP 500 Internal Server Error

**症状**：
```
[18:30:05] 响应状态: 500 Internal Server Error
[18:30:05] 响应内容: Server error
```

**原因**：MiniMax/GLM 服务器错误

**解决方法**：
1. 等待几分钟后重试
2. 切换到另一个模型（MiniMax ↔ GLM）

---

### 错误 4：返回空内容

**症状**：
```
[18:30:06] 提取到的回复:
```

**原因**：API 返回了空内容

**解决方法**：
1. 查看完整的响应数据
2. 检查 `data.choices` 是否存在
3. 检查 `data.choices[0].message.content` 是否存在

---

### 错误 5：回复内容异常

**症状**：
```
[18:30:06] 提取到的回复: undefined / null / [object Object]
```

**原因**：数据格式不对

**解决方法**：
1. 查看完整的响应数据
2. 检查 API 返回的数据结构
3. 可能需要调整代码
