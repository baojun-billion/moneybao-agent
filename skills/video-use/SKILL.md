# 🎬 video-use - 视频生成与应用工作流

## 技能定位

**一站式视频创作流水线** — 从创意到发布的完整工作流，让视频创作像搭积木一样简单。

## 工作流阶段

```
创意构思 → 视频生成 → 质量审核 → 内容优化 → 渠道发布 → 效果追踪
```

### 阶段一：创意构思 💡
**目标**：明确视频方向、风格和核心信息

**输出**：
- 视频创意简报（brief）
- 分镜头脚本（如需要）
- 视觉风格参考

**关键决策点**：
- 目标受众是谁？
- 核心信息是什么？
- 视频时长？
- 风格调性（严肃/活泼/温暖/科技感）？

### 阶段二：视频生成 🎬
**目标**：使用 AI 工具生成初版视频

#### 技术选项：

**选项 A：AI 直接生成（`video_generate`）**
适合：快速原型、创意验证、复杂场景
- `prompt`: 视频描述（中英文皆可）
- `durationSeconds`: 目标时长（建议 15-60s）
- `aspectRatio`: 比例（16:9/9:16/1:1）
- `image/images`: 参考图（可选）
- `audio`: 是否带音频

**技巧**：
- Prompt 要具体：描述场景、动作、氛围、风格
- 参考图能显著提升一致性
- 先生成短版本（15s），确认方向后再扩展

**选项 B：HyperFrames 代码生成**
适合：产品演示、数据可视化、品牌动画、需要精确控制
- **核心优势**：
  - 用 HTML + CSS + JS 写视频，完全可控
  - 特别适合产品 UI 演示、数据图表动画
  - 可复用代码，批量生成变体
  - AI 智能体友好，自动生成代码

- **安装与使用**：
  ```bash
  # 安装 HyperFrames skill（让 AI 学会写视频）
  npx skills add heygen-com/hyperframes
  
  # 初始化项目
  npx hyperframes init my-video
  
  # 渲染视频
  npx hyperframes render
  ```

- **适用场景**：
  - 产品功能演示
  - 数据可视化动画
  - 品牌Logo动画
  - 文字特效视频
  - 系列化内容（同一模板批量生成）

- **技术栈**：
  - HTML：搭建画面
  - Tailwind CSS：控制样式
  - GSAP/Lottie/Three.js：做动画
  - 一行命令渲染成 MP4

#### 选择指南：
```
复杂自然场景 → video_generate（AI 直接生成）
产品/数据/品牌动画 → HyperFrames（代码生成）
快速验证创意 → video_generate
需要精确控制 → HyperFrames
```

### 阶段三：质量审核 👁️
**目标**：评估视频是否达到预期

**检查清单**：
- [ ] 视觉质量是否清晰？
- [ ] 动作是否流畅自然？
- [ ] 色调风格是否符合品牌？
- [ ] 核心信息是否传达？
- [ ] 是否有明显的 AI 瑕疵？

**审核标准**：
- **S 级**：可直接发布，无需修改
- **A 级**：小幅调整即可使用
- **B 级**：需要重新生成
- **C 级**：创意方向需调整

### 阶段四：内容优化 ✨
**目标**：润色和完善视频

**优化方向**：
- 调整 Prompt 重新生成
- 添加字幕/文字叠加
- 背景音乐/配音
- 色彩调优
- 剪辑优化（首尾衔接、转场）

### 阶段五：渠道发布 📤
**目标**：将视频分发到目标平台

**发布前检查**：
- [ ] 视频格式符合平台要求
- [ ] 封面图（thumbnail）已准备
- [ ] 标题和文案已优化
- [ ] 标签（tags）已设置
- [ ] 发布时间已规划

**平台特性**：
- **抖音/TikTok**：竖屏（9:16），节奏快，15-60s
- **视频号**：可横可竖，更偏向真实感
- **小红书**：竖屏，强调美感，配合图文
- **B站**：横屏（16:9），内容深度更高
- **YouTube**：横屏，支持较长内容

### 阶段六：效果追踪 📊
**目标**：收集数据，优化后续创作

**关注指标**：
- 播放量、完播率
- 点赞、评论、分享
- 转化率（如适用）
- 受众反馈

## Prompt 工程技巧

### 基础模板

```
[场景描述]，[动作/动态]，[氛围/情绪]，[风格参考]，[技术细节]
```

### 优质 Prompt 示例

**产品展示（AI 直接生成）**：
```
一个精致的手机产品展示视频，手机在空中缓慢旋转，展示全面屏设计和玻璃背板的质感。光线从侧面扫过，展现金属边框的光泽。背景是极简的渐变色，色调温暖高级。4K 高清画质，慢动作镜头。
```

**产品功能演示（HyperFrames）**：
```
创建一个 30 秒的产品功能演示视频，展示我们的智能家居 App。

画面构成：
1. 手机界面从左侧滑入（0-3s）
2. 点击「一键回家」按钮，界面放大（3-6s）
3. 展示灯光亮起、空调开启的动画效果（6-12s）
4. 镜头切换到房间实景，展示真实的智能设备响应（12-20s）
5. 右上角浮层显示「能耗节省 30%」数据（20-25s）
6. 品牌Logo淡入，配文「让家更智能」（25-30s）

风格要求：
- 使用 Tailwind CSS 极简设计风格
- 主色调：科技蓝 #3B82F6
- 动画使用 GSAP，流畅且可控
- 背景使用微妙的渐变动画
- 文字使用系统字体，清晰易读
```

**品牌故事**：
```
温暖感人的品牌故事视频。从清晨的第一缕阳光开始，创业者在工作台前专注工作的场景。镜头切换到团队协作的画面，每个人脸上洋溢着笑容。最后是产品出现在用户手中的特写，用户满意的表情。整体色调温暖柔和，配乐温馨动人。
```

**社交媒体快闪**：
```
节奏快速的社交媒体视频。一组 15-30s 的画面拼接，展示产品在不同场景下的使用。每个画面 2-3 秒，配合动感的转场效果。色彩鲜艳明亮，充满活力和时尚感。适合 TikTok/抖音平台。
```

**数据可视化动画（HyperFrames）**：
```
创建一个 20 秒的数据增长展示视频，用于投资者汇报。

内容构成：
1. 标题「2024年度用户增长」淡入（0-2s）
2. 柱状图从底部升起，每个柱子代表一个月（2-10s）
3. 数字滚动动画，显示从 10万 到 500万 的增长（10-15s）
4. 折线图叠加，展示趋势曲线（15-18s）
5. 最终数据「500万+ 用户」放大显示（18-20s）

技术要求：
- 使用 Chart.js 或 D3.js 绑定数据
- GSAP 控制动画时间线
- 响应式设计，适配不同尺寸
- 颜色使用品牌渐变色
```

### Prompt 优化清单

- ✅ **具体场景**：不要说"一个产品"，要说"一个黑色的 iPhone 在木桌上"
- ✅ **动作描述**：描述物体的运动（旋转、浮动、滑动）
- ✅ **光影细节**：光线方向、强度、质感
- ✅ **氛围营造**：温暖、科技感、活力、高级
- ✅ **技术规格**：4K、慢动作、景深等
- ✅ **参考风格**：电影质感、动画风格、实拍感

## 质量标准

### 视频质量分级

**S 级（可直接发布）**：
- 视觉质量优秀，无明显 AI 瑕疵
- 动作流畅自然
- 色调风格统一
- 核心信息清晰传达

**A 级（小幅修改）**：
- 整体质量良好
- 有 1-2 个小瑕疵
- 调色或剪辑可优化

**B 级（需重新生成）**：
- AI 瑕疵较多
- 动作不自然
- 风格不统一

**C 级（方向调整）**：
- 核心创意偏离预期
- 需要重新构思方向

### 质量把控原则

1. **不放过明显瑕疵**：AI 生成视频常见问题（手部变形、物体穿模等）必须识别
2. **重视品牌一致性**：色调、风格必须符合品牌调性
3. **关注用户体验**：视频要流畅、易懂、有感染力
4. **追求持续改进**：每次创作后总结经验，优化 Prompt 和流程

## 常见问题

### Q: 生成的视频有 AI 瑕疵怎么办？
**A**：
1. 修改 Prompt，增加更具体的描述
2. 使用参考图引导风格
3. 尝试不同的 aspectRatio 或 duration
4. 如问题严重，考虑重新构思创意

### Q: 如何提高视频的一致性？
**A**：
1. 使用参考图（image/images 参数）
2. 在 Prompt 中强调风格关键词
3. 为系列视频建立固定的 Prompt 模板

### Q: 视频生成失败或质量很差？
**A**：
1. 检查 Prompt 是否过于复杂
2. 尝试缩短视频时长
3. 检查网络连接和 API 状态
4. 联系 jovibao_agent 进行技术排查

### Q: 多平台发布的视频有什么区别？
**A**：
- **抖音/TikTok**：竖屏（9:16），节奏快，15-60s
- **视频号**：可横可竖，偏真实感
- **小红书**：竖屏，强调美感
- **B站**：横屏（16:9），内容深度高
- **YouTube**：横屏，支持较长内容

### Q: 什么时候用 HyperFrames，什么时候用 AI 直接生成？
**A**：
**选择 HyperFrames**：
- 需要精确控制每一帧（产品演示、数据可视化）
- 需要批量生成变体（系列化内容）
- 内容涉及 UI、界面、文字特效
- 需要复用代码（降低成本）

**选择 AI 直接生成**：
- 复杂自然场景（风景、人物、动物）
- 创意探索阶段（快速验证想法）
- 不需要精确控制的创意内容
- 想要更"电影感"的画面

### Q: HyperFrames 学习成本高吗？
**A**：
- **无需从头学**：安装 skill 后，AI 会自动生成代码
- **熟悉的工具**：HTML + Tailwind + GSAP 都是成熟技术
- **渐进式**：先让 AI 生成，再根据需要调整
- **社区支持**：HeyGen 提供开源代码和示例

### Q: HyperFrames 生成的视频质量如何？
**A**：
- **优点**：清晰度极高（4K+），无 AI 瑕疵，动画流畅可控
- **限制**：不适合复杂自然场景（风景、人物、动物）
- **适用**：产品、数据、品牌动画、UI 演示等需要精确控制的内容
